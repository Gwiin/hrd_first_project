# Gamma 발표 PPT 생성 프롬프트

아래 프롬프트를 그대로 `Gamma`에 입력하면 스마트 환경제어 하우스 프로젝트 발표용 PPT 초안을 생성할 수 있다.

```text
다음 내용을 바탕으로 대학교/교육기관 프로젝트 발표용 PPT를 한국어로 만들어줘.

주제는 "스마트 환경제어 하우스"이며, Raspberry Pi Pico 2 W 3대와 MQTT 통신을 활용한 IoT 프로젝트 발표 자료다.

발표 목적:
- 프로젝트의 기획 배경, 시스템 구성, 구현 내용, 테스트 결과, 기대 효과를 명확하게 설명하는 것
- 기술 시연이 가능한 팀 프로젝트 발표 자료로 구성할 것
- 너무 화려하기보다 깔끔하고 전문적인 스타일로 만들 것
- 발표 시간은 약 7~10분 분량으로 맞출 것

디자인 방향:
- 전체적으로 깔끔하고 현대적인 공학/IoT 발표 스타일
- 배경은 밝은 톤, 포인트 컬러는 오렌지/브라운/그린 계열
- 과한 장식보다 구조가 잘 보이게
- 아이콘, 다이어그램, 흐름도 적극 활용
- 텍스트는 너무 많지 않게 핵심 위주로 요약
- 각 슬라이드는 발표자가 설명하기 좋게 구성
- 표, 블록 다이어그램, 노드 구조도, 토픽 흐름도 등을 시각적으로 넣어줄 것

프로젝트 핵심 내용:
- 프로젝트명: 스마트 환경제어 하우스
- Raspberry Pi Pico 2 W 3대를 사용한 분산형 IoT 시스템
- Node A: 조도, 온도, 습도 센서 측정 담당
- Node B: 조명과 환기창 제어 담당
- Node C: 중앙 관리 및 제어 판단 담당
- Web Console: 전체 상태 모니터링 및 수동 제어 담당
- 통신 방식: Wi-Fi + MQTT
- 주요 기능:
  - 실내 환경 실시간 측정
  - 조도 기준 자동 조명 제어
  - 온도/습도 기준 자동 환기 제어
  - AUTO / MANUAL 모드 지원
  - 웹 콘솔을 통한 상태 확인 및 수동 제어
  - heartbeat 기반 연결 상태 확인

자동 제어 기준:
- light < 280 이면 조명 ON
- light > 320 이면 조명 OFF
- temp > 28 또는 humidity > 70 이면 창문 OPEN
- temp <= 27 그리고 humidity <= 65 이면 창문 CLOSE

MQTT 주요 토픽:
- house/env
- house/mode
- house/cmd/light
- house/cmd/window
- house/status/nodeB
- house/status/nodeC
- house/heartbeat/nodeA
- house/heartbeat/nodeB
- house/heartbeat/nodeC

구현 결과:
- Node A가 센서값을 MQTT로 발행
- Node C가 센서 데이터를 받아 자동 제어 판단
- Node B가 실제 LED와 서보모터 동작 수행
- Web Console에서 환경값, 상태, heartbeat, 로그를 확인 가능
- AUTO/MANUAL 모드 전환 가능
- 전체 MQTT 흐름 통합 테스트 완료

하드웨어 구성:
- Raspberry Pi Pico 2 W 3대
- CDS 조도 센서
- DHT 계열 온습도 센서
- WS2812 RGB Strip
- SG90 서보모터
- 브레드보드, 점퍼선

발표 자료를 다음 슬라이드 순서로 구성해줘:

1. 표지
- 프로젝트명
- 팀 프로젝트 발표 자료 느낌
- 스마트 하우스/IoT 느낌의 메인 비주얼

2. 프로젝트 개요
- 무엇을 만드는 프로젝트인지
- 왜 이 주제를 선택했는지 간단히 설명

3. 개발 목표
- 자동 환경 제어
- 분산형 노드 구조 구현
- MQTT 통신 학습
- 웹 모니터링 구현

4. 시스템 전체 구성도
- Node A, Node B, Node C, Web Console 관계를 한눈에 보여주는 다이어그램
- MQTT 중심 흐름 강조

5. 노드별 역할 소개
- Node A / Node B / Node C / Web Console
- 표 또는 카드 형태로 정리

6. 하드웨어 및 개발 환경
- 사용 장비와 기술 스택 정리
- Pico 2 W, 센서, 서보, MQTT, Flask 등

7. 주요 기능
- 환경 측정
- 자동 조명 제어
- 자동 환기 제어
- 수동 제어
- 상태 모니터링

8. 자동 제어 로직
- light, temp, humidity 기준을 표와 조건 블록으로 시각화
- if-then 구조가 잘 보이게

9. MQTT 통신 구조
- 주요 토픽과 메시지 흐름 설명
- Node A -> Node C -> Node B 흐름 강조

10. Web Console 소개
- 웹 화면에서 어떤 정보를 보여주는지
- 수동 제어 기능 설명
- 가능하면 대시보드 느낌의 슬라이드

11. 구현 및 테스트 결과
- 단위 테스트
- 통합 테스트
- 전체 동작 확인 결과
- PASS 형태로 깔끔하게 정리

12. 기대 효과 및 확장 방향
- 스마트 홈 구조 학습
- IoT 시스템 통합 경험
- OLED/LCD 추가, 모바일 제어, 데이터 저장 등 확장 가능성

13. 결론
- 프로젝트 요약
- 완성한 핵심 성과 정리
- 발표 마무리 문구

추가 요구사항:
- 각 슬라이드 제목과 핵심 bullet을 한국어로 작성
- 발표자가 읽기 쉽게 너무 긴 문장 대신 짧고 명확한 문장 사용
- 도식화가 가능한 부분은 최대한 시각 중심으로 구성
- 표지와 전체 디자인은 학생 팀 프로젝트 발표 같으면서도 완성도 있게 보여야 함
- 마지막 슬라이드는 “감사합니다” 스타일이 아니라 결론 중심으로 만들어줘
```
